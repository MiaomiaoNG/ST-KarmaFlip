# ST-api-switcher 设计说明

## 目标

本插件用于 SillyTavern 正文生成前自动切换 API。它支持固定模式和随机模式，随机模式按 API 池成员权重抽取单个 API，用于打散模型输出惯性并降低单个模型连续出错的影响。

## 触发范围

正文生成包括普通发送、Swipe、Continue、Regenerate。插件只在文本生成请求前生效，不修改 prompt，不解析模型回复，不导出 API 配置。

## API 来源

第一版只支持手动填写 OpenAI-compatible API，字段包括名称、URL、Key、模型。插件不导入酒馆 Connection Profiles，不读取或修改 `settings.json` / `secrets.json`。

插件配置保存在 `extension_settings.STApiSwitcher`，通过 SillyTavern 的设置保存机制持久化。Key 会保存在插件配置中。

## 池与模式

第一版使用全局默认池，不做按角色或聊天分别指定池。固定模式按条目顺序和单轮运行次数轮询。随机模式从池里按规则抽取一个成员。

## 权重

权重、冷却回合、保底回合、运行次数必须是数字。数字以外输入自动归 0。内部权重按整数处理，小数四舍五入。API 是否参与由 `已启用/已停用` 控制。

当保存时检测到权重大于 0 的成员权重差距过大，弹窗询问：`当前权重差距较大，是否允许插件按照API数量自动均等权重？`。确认后仅将 `weight>0` 的成员权重改为 1；取消则不修改。

## 冷却

冷却采用回合冷却。成员命中后，在接下来 N 次正文生成中不参与随机。允许冷却值大于池内启用 API 数量。如果所有有效成员都在冷却，则所有冷却清零并重新按权重抽取。

## 失败处理

切换失败时重新随机，失败 API 不进入冷却。API 生成失败时该 API 不进入冷却并累计连续失败次数。连续失败 3 次后自动停止该 API 调用，并弹窗提示：`API模型（名字）已连续失败调用3次，已停止该API的调用，请到后台检查设置`。

## 实现优先级

使用极简 fetch 文本生成请求拦截，只拦截文本生成路由，不拦截绘图和语音，不修改 prompt 内容。发送时将请求体 patch 为 OpenAI-compatible reverse proxy：`chat_completion_source=openai`、`reverse_proxy=URL`、`proxy_password=Key`、`model=模型`。
